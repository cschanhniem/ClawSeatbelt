# Product Positioning

## Category Statement

ClawSeatbelt is the local-first OpenClaw security plugin for people who want prompt-injection risk reduction, skill supply-chain inspection, transcript hygiene, and one clear posture report without needing a hosted control plane.

This is the sharpest category frame:

**ClawSeatbelt makes OpenClaw feel safe enough to run every day.**

## Primary Search Intents

These are the search intents the public copy should satisfy naturally:

- OpenClaw security plugin
- OpenClaw prompt injection protection
- OpenClaw skill scanner
- OpenClaw transcript redaction
- OpenClaw tool-call guardrails
- OpenClaw posture report
- OpenClaw plugin security audit
- OpenClaw supply chain security

## Core Promise

- It reduces risk before the model sees dangerous content.
- It blocks obvious high-risk tool flows when policy says it should.
- It cleans sensitive material before transcripts become permanent.
- It inspects skills before trust expands quietly.
- It turns scattered OpenClaw security settings into one readable operator story.

## Differentiators That Matter

### Local By Default

No account, no quota, no server dependency, no cloud service in the hot path.

### Built For Operators

Findings should read like a tight incident brief:

- what happened
- why it matters
- what to do next

### Supply-Chain First

Most products wait until runtime. ClawSeatbelt also cares about the moment before installation, where bad trust decisions become permanent.

### OpenClaw-Native

ClawSeatbelt should amplify:

- `openclaw security audit`
- tool policy
- exec approvals
- pairing
- plugin allowlists

It should not pretend these controls do not exist.

## Messaging Architecture

### Homepage Headline

ClawSeatbelt is the OpenClaw security plugin for local-first trust.

### Homepage Subhead

Score risky messages, block dangerous tool flows, scan skills before trust expands, redact sensitive transcript material, and turn OpenClaw security settings into one calm posture report.

### One-Sentence Pitch

ClawSeatbelt helps OpenClaw users run local agents with fewer blind spots and less operational drag.

### Proof Points

- local-first baseline with no hot-path cloud dependency
- small, inspectable package surface
- explicit composition with native OpenClaw security controls
- skill scanning and transcript hygiene in the same trust layer
- benchmark-driven roadmap instead of vague firewall language

## Why Users Switch

- They want an OpenClaw security plugin without account setup.
- They want something easier to trust than server-bound policy systems.
- They want better visibility into risky skills, not only risky tool calls.
- They want outputs that make operational decisions faster.
- They want a plugin that works with OpenClaw’s built-ins instead of fighting them.

## Audience Fit

### Solo Operators

Give them a default trust layer that is calm, local, and readable.

### Security-Conscious Builders

Give them inspectable artifacts, explainable findings, and a tight threat model story.

### Teams Growing Past Hobby Use

Give them a baseline that is useful alone and still interoperable later with heavier policy systems.

## Messaging Guardrails

- Never say prompt injection is solved.
- Never imply the product is a sandbox or kernel boundary.
- Never hide tradeoffs behind a security slogan.
- Never overclaim benchmark results.
- Keep the tone exact, grounded, and useful.
